# npm: feeding a dependency SBOM via `--sbom`

`sbom-quality` bundles no Node toolchain. For npm you **bring your own** CycloneDX
dependency SBOM, generated by your build, and hand it to the pipeline with `--sbom`
(mutually exclusive with `--go-mod`; `--image` optional). The pipeline then enriches,
augments, quality-patches, and validates it into one CycloneDX 1.6 document — the same
path Go and Java take, so npm is a docs + CI job, not a code change.

Recommended: **`@cyclonedx/cyclonedx-npm`**. It defaults to CycloneDX 1.6 (no flag
needed), stamps `acknowledgement: declared` on licenses, and records each dependency's
npm `integrity` digest. cdxgen (`-t javascript`) is a documented fallback for polyglot
builds.

Measured post-pipeline sbomqs (cyclonedx-npm): solo `--sbom` ≈ **8.2**.

## npm — `@cyclonedx/cyclonedx-npm`

cyclonedx-npm reads the installed `node_modules` tree, so install first, then generate.
No `package.json` change is needed — run it via `npx`:

```sh
npm ci                              # or `npm install`
npx @cyclonedx/cyclonedx-npm --output-format JSON --output-file bom.json
# -> bom.json  (native CycloneDX 1.6, acknowledgement: declared)

sbom-quality --sbom bom.json --supplier-name "ACME" > sbom.cdx.json
```

GitHub Actions:

```yaml
- uses: actions/setup-node@v4
  with: { node-version: "22" }
- name: Build dependency SBOM
  run: |
    npm ci
    npx @cyclonedx/cyclonedx-npm --output-format JSON --output-file bom.json
- name: Quality SBOM
  run: |
    docker run --rm -v "$PWD":/work ghcr.io/nissessenap/sbom-quality:latest \
      --sbom /work/bom.json --supplier-name "ACME" > sbom.cdx.json
```

> **integrity hashes:** cyclonedx-npm parks each dependency's `integrity` digest on a
> `distribution` externalReference, not on the component itself — where sbomqs's checksum
> score can't see it. The pipeline's quality-patch stage lifts those digests onto the
> components (faithful: the same digest, relocated), which is what carries the score from
> ~6.8 to ~8.2. Nothing to configure; it happens automatically.

## cdxgen fallback (polyglot / monorepo builds)

[cdxgen](https://github.com/CycloneDX/cdxgen) is one build-agnostic generator covering
npm alongside other ecosystems. Use it when cyclonedx-npm doesn't fit. It works through
`--sbom`, but **defaults to CycloneDX 1.7** — pin `--spec-version 1.6` to match the
pipeline:

```sh
cdxgen -t javascript --spec-version 1.6 -o cdxgen.bom.json .
sbom-quality --sbom cdxgen.bom.json --supplier-name "ACME" > sbom.cdx.json
```
